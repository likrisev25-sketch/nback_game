# Логика начисления очков

## 📊 НОВАЯ СИСТЕМА (активная с последнего обновления)

### Правила начисления:

| Действие игрока | Совпадение? | Очки | Ошибки | Комментарий |
|----------------|-------------|------|--------|-------------|
| Нажал кнопку ✅ | Да (isMatch=true) | +1 | 0 | **Правильное активное нажатие** |
| Нажал кнопку ❌ | Нет (isMatch=false) | 0 | +1 | Ложное нажатие |
| Не нажал ⏭️ | Да (isMatch=true) | 0 | +1 | **Пропуск совпадения** |
| Не нажал ✓ | Нет (isMatch=false) | 0 | 0 | Правильное бездействие |

### Ключевые изменения:

1. **Очки начисляются ТОЛЬКО за активное нажатие на совпадение**
   - `playerAnswer === true && isMatch === true` → correctAnswers++

2. **Ошибки начисляются за неправильные действия**
   - `playerAnswer === true && isMatch === false` → errors++ (ложное нажатие)
   - `playerAnswer === false && isMatch === true` → errors++ (пропуск совпадения)

3. **Правильное бездействие не начисляет очки**
   - `playerAnswer === false && isMatch === false` → ничего (0 очков, 0 ошибок)

4. **Убраны авто-ответы**
   - Клиент больше не отправляет автоматические ответы за игрока
   - Меньше запросов к серверу
   - Игрок контролирует свои действия

### Пример для N=2:

Последовательность: [1, 2, 1, 3, 2, 3]

| Шаг | Позиция | Совпадение? | Игрок нажал? | Результат |
|-----|---------|-------------|--------------|-----------|
| 0 | 1 | Нет (шаг < N) | Нет | 0 очков, 0 ошибок |
| 1 | 2 | Нет (2≠1) | Нет | 0 очков, 0 ошибок |
| 2 | 1 | **Да** (1=1) | **Да** ✅ | **+1 очко** |
| 3 | 3 | Нет (3≠2) | Нет | 0 очков, 0 ошибок |
| 4 | 2 | Нет (2≠1) | Нет | 0 очков, 0 ошибок |
| 5 | 3 | **Да** (3=3) | Нет ❌ | **+1 ошибка** (пропуск) |

**Итого:** 1 очко, 1 ошибка

## 🤖 Логика для бота

Бот следует тем же правилам:

1. Бот "видит" текущую позицию и историю
2. Бот определяет, есть ли совпадение
3. С вероятностью `accuracy` бот действует правильно:
   - Если accuracy=0.8 → 80% шанс правильного действия
4. Правильное действие:
   - При совпадении → нажать кнопку
   - Без совпадения → не нажимать
5. Очки начисляются по тем же правилам

### Пример для бота с accuracy=80%:

| Шаг | Совпадение? | Бот нажал? | accuracy | Результат |
|-----|-------------|------------|----------|-----------|
| 1 | Нет | Нет | 80% | 0 очков, 0 ошибок ✓ |
| 2 | Да | Да ✅ | 80% | +1 очко ✓ |
| 3 | Нет | Да ❌ | 20% (ошибка) | +1 ошибка |
| 4 | Да | Нет ❌ | 20% (ошибка) | +1 ошибка |

## 🔧 Технические детали

### API endpoint: `/api/game/[sessionId]/answer`

**Логика обновления статистики:**

```typescript
// Очки ТОЛЬКО за активное нажатие на реальное совпадение
if (playerAnswer === true && isMatch === true) {
  await db.update(gamePlayers)
    .set({ correctAnswers: sql`${gamePlayers.correctAnswers} + 1` })
    .where(eq(gamePlayers.id, playerIdStr));
}

// Ошибка за любое неправильное действие
if (!isCorrect) {
  await db.update(gamePlayers)
    .set({ errors: sql`${gamePlayers.errors} + 1` })
    .where(eq(gamePlayers.id, playerIdStr));
}
```

## 📈 Сравнение систем

### Старая система:
- ✅ Начислялись очки за "правильное бездействие"
- ❌ Игрок получал много очков пассивно
- ❌ Авто-ответы создавали нагрузку на сервер
- ❌ Бот получал очки за каждое правильное бездействие

### Новая система:
- ✅ Очки только за активные правильные действия
- ✅ Честная система scoring
- ✅ Меньше запросов к серверу
- ✅ Бот следует тем же правилам
- ✅ Пропуск совпадения = ошибка (как в классическом N-back)

## 🎮 Стратегия игры

Для максимального счета:
1. Внимательно следите за последовательностью
2. Нажимайте кнопку **только** при уверенности в совпадении
3. Не нажимайте "на всякий случай" - это ошибка
4. Не пропускайте явные совпадения - это тоже ошибка

**Оптимальная стратегия:**
- Высокая точность > много нажатий
- Лучше пропустить сомнительное совпадение, чем сделать ложное нажатие
