# Система онлайн-турниров

## Описание

Добавлена полная система онлайн-турниров с поддержкой мультиплеера в реальном времени через WebSocket (Socket.IO).

## Новые возможности

### 1. Создание турниров
- Пользователи могут создавать турниры с настройками:
  - Название
  - N-значение (1-4)
  - Количество раундов (3, 5, 10)
  - Шагов в раунде (20, 30, 50)
  - Скорость (2500ms, 2000ms, 1500ms)
  - Минимум/максимум игроков (2-6)
  - Пароль (опционально)

### 2. Список доступных турниров
- Страница `/tournaments` показывает все активные турниры со статусом 'waiting'
- Автоматическое обновление списка каждые 10 секунд
- Показывает текущее количество игроков, настройки и участников

### 3. Присоединение к турниру
- Пользователи могут присоединяться к существующим турнирам
- Отображение всех игроков в турнире
- Статус готовности каждого игрока

### 4. Запуск турнира
- Только хост (создатель) может запустить турнир
- Турнир запускается когда:
  - Все игроки готовы
  - Достигнуто минимальное количество игроков
- Обратный отсчет перед началом (5 секунд)

### 5. Real-time обновления через WebSocket
- Присоединение/выход игроков
- Изменение статуса готовности
- Обратный отсчет
- Запуск игры

## Структура файлов

### Frontend
- `app/tournaments/page.tsx` - Список доступных турниров
- `app/tournament/page.tsx` - Перенаправление на /tournaments
- `app/tournament/[id]/page.tsx` - Страница конкретного турнира

### Backend API
- `app/api/tournament/list/route.ts` - Получение списка турниров
- `app/api/tournament/create/route.ts` - Создание турнира
- `app/api/tournament/join/route.ts` - Присоединение к турниру
- `app/api/tournament/[tournamentId]/route.ts` - Получение данных турнира

### База данных
- `drizzle/0001_tournament_tables.sql` - Миграция для таблиц турниров
- `db/schema/index.ts` - Схемы `tournaments` и `tournament_players`

### WebSocket
- `lib/socket-server.ts` - Обработчики событий турниров

## WebSocket Events

### Клиент → Сервер
- `tournament:join` - Присоединение к турниру
- `tournament:leave` - Выход из турнира
- `tournament:ready` - Изменение статуса готовности
- `tournament:start` - Запуск турнира (только хост)

### Сервер → Клиент
- `tournament:update` - Обновление данных турнира
- `tournament:player-joined` - Игрок присоединился
- `tournament:player-left` - Игрок покинул турнир
- `tournament:player-ready` - Изменение статуса готовности
- `tournament:countdown` - Обратный отсчет
- `tournament:start-game` - Запуск игры
- `tournament:error` - Ошибка

## Использование

### Создание турнира

1. Перейдите на `/tournaments`
2. Нажмите "Создать турнир"
3. Заполните форму
4. Нажмите "Создать турнир"
5. Вас перенаправит на страницу турнира

### Присоединение к турниру

1. Перейдите на `/tournaments`
2. Выберите турнир из списка
3. Нажмите "Присоединиться"
4. Вас перенаправит на страницу турнира

### Запуск турнира

1. Дождитесь, пока все игроки нажмут "Готов"
2. Хост нажимает "Начать турнир"
3. Начинается обратный отсчет (5 секунд)
4. Игра запускается автоматически

## Обновление базы данных

Запустите миграцию:

```bash
cd nback-game
npx drizzle-kit push
```

Или вручную выполните SQL из `drizzle/0001_tournament_tables.sql`.

## Примечания

- Для работы турниров необходимо запущен Socket.IO сервер
- Socket.IO инициализируется в `server.js` через `initSocket()`
- Требуется авторизация пользователя
- Минимум 2 игрока для начала турнира
