# 🔧 Исправление создания лобби - Инструкция

## Проблема
Создание лобби не работало из-за:
1. Отсутствия базы данных
2. Проблем с авторизацией

## Решение

### 1. База данных

Проект теперь поддерживает **два варианта** базы данных:

#### Вариант A: PostgreSQL (Neon) - Рекомендуемый
```bash
# Добавьте в .env:
DATABASE_URL=postgresql://user:password@host:5432/database
```

#### Вариант B: SQLite (локальный) - Для тестирования
Если DATABASE_URL не установлен или подключение не удалось, автоматически используется SQLite файл `nback.db`

### 2. Авторизация

Временно упрощена для тестирования:
- userId создаётся автоматически через `nanoid()`
- userName передаётся из клиента

### 3. Игра с ботом

Добавлена поддержка ботов в лобби:
- Чекбокс "Добавить бота" в форме создания
- Настройка имени бота
- Настройка точности (50-100%)
- Бот автоматически готов к игре

## Запуск

### Шаг 1: Установка зависимостей
```bash
cd nback-game
npm install
```

### Шаг 2: Проверка .env
```bash
# Проверьте что файл .env существует
cat .env

# Если нет, создайте:
DATABASE_URL=postgresql://postgres:password@localhost:5432/nback_db
BETTER_AUTH_SECRET=your-secret-key
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Шаг 3: Запуск
```bash
npm run dev
```

Откройте http://localhost:3000

## Тестирование

### Создание лобби с ботом:
1. Перейдите на `/lobbies`
2. Нажмите "+ Создать лобби"
3. Заполните форму
4. Поставьте галочку "Добавить бота"
5. Настройте имя и точность бота
6. Нажмите "Создать"
7. Вы увидите бота в списке игроков

### Проверка работы:
```bash
# Проверьте логи в консоли браузера:
🔵 [LobbyList] Создание лобби для пользователя: ...
🔵 [lobby/create] POST request received
✅ [lobby/create] Lobby created successfully: ...

# Проверьте консоль сервера:
✅ [db] Connected to local SQLite (nback.db)
```

## Если что-то не работает

### Ошибка подключения к БД:
```bash
# Проверьте что SQLite установлен
npm list better-sqlite3

# Если нет, установите:
npm install better-sqlite3 --save
```

### Ошибка создания лобби:
```bash
# Проверьте логи сервера:
# Должно быть: ✅ [lobby/create] Lobby created successfully

# Проверьте логи браузера (F12)
# Должны быть логи с 🔵
```

### Лобби не появляется в списке:
```bash
# Убедитесь что статус лобби = 'waiting'
# Проверьте таблицу lobbies в базе данных
```

## Известные ограничения

1. **Авторизация** - временно упрощена, userId создаётся случайно
2. **Боты** - пока не играют, только отображаются в списке
3. **Турниры** - игра в турнирах пока не реализована
