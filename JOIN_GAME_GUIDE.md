# 🎮 Руководство по подключению к игре через ID

## 📋 Как это работает

Каждая игра получает **уникальный UUID ID** при создании. Этот ID можно передать друзьям, и они смогут присоединиться к вашей игре.

---

## 🔧 Техническая реализация

### 1. Генерация уникального ID игры

При создании игры в `/api/game/create/route.ts`:

```typescript
const sessionId = uuidv4(); // Генерирует уникальный UUID
```

**Пример ID:** `ab334218-bdee-4c82-b099-3af4efb89419`

### 2. Создание игры

**API Endpoint:** `POST /api/game/create`

**Request Body:**
```json
{
  "name": "Моя игра",
  "nValue": 3,
  "totalSteps": 30,
  "baseSpeedMs": 2000,
  "maxPlayers": 4,
  "addBot": false,
  "botAccuracy": 80
}
```

**Response:**
```json
{
  "success": true,
  "sessionId": "ab334218-bdee-4c82-b099-3af4efb89419",
  "playerId": "4f961470-734a-4942-b212-f3f8b253848a",
  "name": "Моя игра",
  "nValue": 3,
  "baseSpeedMs": 2000,
  "status": "waiting"
}
```

### 3. Присоединение к игре

**API Endpoint:** `POST /api/game/join`

**Request Body:**
```json
{
  "sessionId": "ab334218-bdee-4c82-b099-3af4efb89419",
  "playerName": "Игрок2"
}
```

**Response:**
```json
{
  "success": true,
  "playerId": "new-uuid-here",
  "playerName": "Игрок2",
  "playerCount": 2,
  "maxPlayers": 4
}
```

### 4. Проверки при присоединении

API `/api/game/join` выполняет следующие проверки:

1. ✅ **Валидация sessionId** - должен быть корректным UUID
2. ✅ **Валидация playerName** - не пустое, максимум 50 символов
3. ✅ **Проверка существования сессии** - игра существует в БД
4. ✅ **Проверка статуса** - игра в статусе `waiting`
5. ✅ **Проверка лимита игроков** - `playerCount < maxPlayers`

### 5. Ошибки при присоединении

| Ошибка | Код | Описание |
|--------|-----|----------|
| `Некорректный ID сессии` | 400 | sessionId не является валидным UUID |
| `Введите имя игрока` | 400 | playerName пустой или некорректный |
| `Сессия не найдена` | 404 | Игра с таким ID не существует |
| `Игра уже началась` | 400 | Статус сессии не `waiting` |
| `Игра полна` | 400 | `playerCount >= maxPlayers` |

## 🚀 Как использовать

### Шаг 1: Создание игры

1. Откройте `http://localhost:3000`
2. Введите **ваше имя**
3. Введите **название игры**
4. Настройте параметры (N-значение, количество игроков, бота)
5. Нажмите **"Создать игру"**

### Шаг 2: Получение ID игры

После создания игры появится блок с ID:

```
Игра создана!
ID игры:
ab334218-bdee-4c82-b099-3af4efb89419  [📋]
```

Нажмите **📋** чтобы скопировать ID, или скопируйте вручную.

### Шаг 3: Передача ID другу

Отправьте ID игры другу любым способом:
- Мессенджер
- Email
- Голосовой чат
- Любой другой способ

### Шаг 4: Присоединение к игре

**Друг открывает `http://localhost:3000`:**

1. Вводит **своё имя**
2. Вставляет **ID игры** в поле "ID игры"
3. Нажимает **"Присоединиться"**

### Шаг 5: Начало игры

После присоединения всех игроков:
- Создатель нажимает **"Начать игру"**
- Все игроки попадают в одну игру

---

## 📊 Обновление playerCount

После каждого присоединения:

1. **Сервер** обновляет количество игроков в БД
2. **API `/api/game/list`** возвращает актуальное количество
3. **UI** показывает `playerCount/maxPlayers`
4. Кнопка **"Войти"** становится неактивной когда `playerCount >= maxPlayers`

## ⚠️ Важные замечания

1. **ID игры уникален** - каждая игра получает новый UUID
2. **Игра в статусе `waiting`** - нельзя присоединиться если игра уже началась
3. **Лимит игроков** - нельзя присоединиться если игра полна
4. **Имя игрока обязательно** - нельзя присоединиться без имени

## 🛠️ API Reference

### POST /api/game/create

**Создаёт новую игровую сессию**

**Request:**
```json
{
  "name": string,
  "nValue": number (1-5),
  "totalSteps": number (10-100),
  "baseSpeedMs": number (500-5000),
  "maxPlayers": number (2-4),
  "addBot": boolean,
  "botAccuracy": number (50-100)
}
```

**Response:**
```json
{
  "success": true,
  "sessionId": string,
  "playerId": string,
  "name": string,
  "nValue": number,
  "baseSpeedMs": number,
  "status": "waiting"
}
```

---

### POST /api/game/join

**Присоединяет игрока к существующей сессии**

**Request:**
```json
{
  "sessionId": string (UUID),
  "playerName": string (1-50 chars)
}
```

**Response:**
```json
{
  "success": true,
  "playerId": string,
  "playerName": string,
  "playerCount": number,
  "maxPlayers": number
}
```
